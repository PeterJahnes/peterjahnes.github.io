Socks in an Array?

  • Peter Jahnes
  • 10/7/2014
  • Arrays and Hashes, a comparison

Ruby has Array's and Hash'es. They are both groups of objects.

[ 4, 2, 5, 3, 7, 10 ]

{ 0 => 4, 1 => 2, 0 => 5, 3 => 3, 4 => 7, 5 => 10 }

They can both have any type of object such as an integer above or a string below,

[ 'cat', 'dog', 'mouse']

{ 'feline' => 'cat', 'k-9' => 'dog', 'rodent' => 'mouse' }

and even different types of objects and even out of order.

['cat', 4, 'dog', 7, 'mouse', 10]

{ 'feline' => 2, 0 => 'dog', 'rodent' => 7 }

I know them both from another coding language, PHP. The names for them in php are more descriptive.

An array in ruby is an Indexed array in PHP. A Ruby array is a list of things that have an index, kinda like a very simple address, starting at 0 and counting up. So if I wanted something from the array, I would say

my_cool_array[42]

and it would return

Socks

The fact that they have an index starting from 0 incrementing by 1 to whatever, is not visibly apparent but it is operatively apparent.

A hash in ruby is an Associative array in PHP. A Ruby hash is like a list that has labels. So instead of having to go to address 42, we can go to the label like so,

my_cool_hash['My favorite Christmas present']

and it would return,

Socks

Both Hashes and Arrays are very useful in ruby for dealing with data. It can allow you to sort, remove, add, change, and numerous other actions that can be very helpful in programing.
