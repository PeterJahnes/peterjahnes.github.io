Class vs Module

  • Peter Jahnes
  • 11/1/2014
  • Simply Put
<

Classes and modules are very similar in many ways but there are definitely specific uses for both of them. Both have methods and constants inside of them. They have basically the same syntax. However, you should treat module like a hardware store and classes like a kitchen.

<

Modules are used to group a collection of methods, constants, and classes. Modules do not allow inheritance like classes do instead you include it in one or many classes and other methods. It allows you to have DRYer code by factoring out objects that are commonly used. In a way it is like having global but selective inheritance. You can have a class inside a module but not the other way around ( at least you wouldn't want to). Modules can not have instances, but they can hold classes. The reason I say use them like hardware store is because you can use them to include building materials and tools for what you are doing.

<

A classes best use is for instances. This allows you to create objects and manipulate them in a standard way as you see fit. You can use them to have inheritance so that they inherit from another class, this allow you to make your code DRYer if you have many related but not identical objects that you want to make. You don't have to use them instanced, which means that you can use them as a module and take advantage of the inheritance in your organization.

<

Which one you use in the end with vary greatly with the scope and organization of your project. Being that I am not experienced in ruby I can't tell you which one is used more, but I can see plenty of use for both.
